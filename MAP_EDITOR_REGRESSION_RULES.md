# 地图编辑器回归守则

这份文档用于约束 `Axe2DEditor` 地图编辑器的后续修改，目标是避免把已经稳定的功能改崩。

## 适用范围

- `Editor/Controls/MapCanvasPanel.cs`
- `Editor/Modules/MapEditorForm.cs`
- `Editor/Controls/TilesetPalettePanel.cs`
- `Editor/Modules/TilesetPlannerDialog.cs`
- `Core/Maps/RpgMakerAutoTile.cs`

## 必须遵守

- A1 C 海洋装饰和 E 瀑布的语义固定为：默认不按 Shift 是覆盖模式，按住 Shift 是自动地形模式。
- A1 的 C 和 E 不能共用同一套补齐表。C 用 2x3 地面 quarter 表，E 用 2x1 瀑布 quarter 表。
- A3 的语义固定为：默认是自动墙体拼接，按住 Shift 只写入单格固定形态，不得退化成 2x2 地图占位，也不得走 A1/A2 的自动逻辑。
- A2 的地面自动元件逻辑不要因为 A3 或快捷键改动而被连带重写，除非这次任务明确要求。
- Shift、Alt、吸管、黄色选框、自动地形、工具切换之间必须彼此隔离，不允许一个快捷键改动另一个模式的状态。
- 画笔尺寸和橡皮尺寸是两套独立状态，不允许再共用一个尺寸值。
- 图集规划器、图集面板、地图绘制和地形生成必须同时对齐同一套规则，不能只改一个入口。
- UI 文案默认必须走本地化链路，不能在界面里暴露不该出现的英文 key 或内部字段名。
- WinForms 布局优先保证文字完整显示，不要为了补逻辑而顺手破坏对齐、间距、边框或默认风格。

## 修改前检查

1. 先确认要改的是哪条路径，是 A1、A2、A3、快捷键、尺寸，还是 UI 布局。
2. 先 `rg` 搜索所有相关引用，再动代码。
3. 只改当前目标分支，不要顺手给旧逻辑再叠一层条件。
4. 修改快捷键时，先检查是否会影响选区、自动地形、图集高亮和绘制模式。
5. 修改后至少构建一次，确认没有把别的路径一起改坏。

## 已知高风险点

- A1 C/E 的 Shift 语义最容易反向。
- A3 最容易被误写成 2x2 贴图块，或者和 A2 的自动元件逻辑混在一起。
- Alt 吸管最容易误伤选区高亮和当前工具状态。
- 画笔尺寸最容易和橡皮尺寸、状态栏、预览面板共用成一份状态。
- 图集规划最容易只改生成逻辑，忘记同步左侧折叠预览和地图选取逻辑。

## 当前实现约定

- A3 固定形态使用单格存储，必要时通过标签记录固定 shape，再由渲染读取。
- A1 覆盖模式使用 `TileX` / `TileY` 和 `a1Overlay:*` 标签。
- A1 自动地形模式使用 `TerrainId`。
- 任何新快捷键都必须先证明它不会改变上述约定。
